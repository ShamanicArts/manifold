## Issue Description

**Status:** REGRESSION

The blend tab "Pitch" knob (`blendSamplePitch`) is intended to transpose whichever source is **not** being key-tracked, allowing users to set the root note of the locked source relative to the key-tracking source.

**Intended Behavior:**
- **Wave mode** (blendKeyTrack=0): Wave keytracks, Sample locked. Pitch knob transposes Sample.
- **Sample mode** (blendKeyTrack=1): Sample keytracks, Wave locked. Pitch knob transposes Wave.
- **Both mode** (blendKeyTrack=2): Both keytrack. (Pitch knob behavior TBD)

**Current Broken Behavior:**
The pitch knob **only affects the sample** regardless of which key-track mode is selected. When in "Sample" mode (where the wave should be the locked source transposable by the pitch knob), the wave ignores the pitch knob entirely.
