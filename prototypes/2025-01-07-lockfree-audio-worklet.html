Interactive demonstration of Single-Producer Single-Consumer lock-free queues for real-time audio, matching the pattern being implemented in Manifold's ControlServer.h.

Why Lock-Free for Audio?

In the Manifold codebase, the SPSCQueue and EventRing handle communication between the non-real-time control thread and the real-time audio thread. Locks can cause:

  • Priority inversion - Control thread blocks audio thread
  • Non-deterministic latency - Lock contention causes glitches
  • Dropouts - Audio callback misses its deadline

Lock-free algorithms guarantee progress - the audio thread never waits.

The Pattern (from Manifold ControlServer.h)

// Single-Producer Single-Consumer Queue template<std::size_t Capacity> class SPSCQueue { public: // Called from producer thread (control/gui) bool enqueue(const ControlCommand& cmd) { const std::size_t w = writeIdx.load(std::memory_order_relaxed); const std::size_t next = (w + 1) % Capacity; // Full check (acquire ensures we see latest readIdx) if (next == readIdx.load(std::memory_order_acquire)) return false; // Queue full ring[w] = cmd; writeIdx.store(next, std::memory_order_release); // Publish return true; } // Called from consumer thread (audio callback) bool dequeue(ControlCommand& cmd) { const std::size_t r = readIdx.load(std::memory_order_relaxed); // Empty check (acquire ensures we see latest writeIdx) if (r == writeIdx.load(std::memory_order_acquire)) return false; // Queue empty cmd = ring[r]; readIdx.store((r + 1) % Capacity, std::memory_order_release); return true; } private: std::array<ControlCommand, Capacity> ring{}; alignas(64) std::atomic<std::size_t> writeIdx{0}; alignas(64) std::atomic<std::size_t> readIdx{0}; };

❌ Old Pattern (int indices)

// Risk: Negative indices wrap to huge values std::atomic<int> writeIdx{0}; // Danger: If value becomes negative... if (next == readIdx.load()) // Comparison with negative is UB land
  • Signed overflow is UB
  • Negative indices possible
  • Implicit conversions everywhere

✅ New Pattern (size_t indices)

// Safe: Unsigned wraps predictably std::atomic<std::size_t> writeIdx{0}; // Safe: Always 0 to Capacity-1 const std::size_t next = (w + 1) % Capacity;
  • Well-defined overflow (modulo)
  • No negative values possible
  • Explicit conversions at boundaries

Memory Ordering Explained

Ordering Usage in Manifold Why
memory_order_relaxed Reading own index Thread only writes its own index, no synchronization needed
memory_order_acquire Reading other index Synchronizes-with the release store in other thread
memory_order_release Updating own index Publishes data to other thread (happens-before)

Integration with Manifold

This exact pattern is used in Manifold for:

  • ControlServer → Audio: Parameter changes, play/stop commands
  • Audio → ControlServer: Event notifications (EVENT messages)
  • MIDI → Audio: Note on/off, CC messages

The conversion from int to std::size_t indices eliminates a class of potential bugs where negative values could wrap to huge indices, causing memory corruption or crashes.
