Integration Notes

How this fits into app.ts:

Add a new widgetType === "scope" branch in buildControl(). The widget receives an endpoint path like /audio/scope/left and renders a canvas that updates from either SSE, bundle polling, or simulated data.

The canvas uses devicePixelRatio scaling for crisp lines on Retina displays. The render loop is driven by requestAnimationFrame and consumes the latest Float32Array samples stored on the widget state.

if (widgetType === "scope") { const canvas = document.createElement("canvas"); const dpr = window.devicePixelRatio || 1; canvas.width = (widget.w || 320) * dpr; canvas.height = (widget.h || 120) * dpr; canvas.style.width = (widget.w || 320) + "px"; canvas.style.height = (widget.h || 120) + "px"; const ctx = canvas.getContext("2d"); ctx.scale(dpr, dpr); startScopeRenderLoop(widget, canvas, ctx, target, endpoint); controlCard.append(canvas); return controlCard; }

OSCQuery Streaming Recommendations

  • SSE is the sweet spot for scopes: one HTTP connection, server pushes frames at 30–60 Hz.
  • Base64-encoded Float32 is ~3× faster to parse than JSON number arrays.
  • Decimate on the server: only send 2× the canvas width in samples.
  • Mark endpoints hidden from the generic parameter list (like /stats today).
